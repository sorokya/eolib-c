Number Codec

EO encodes integers as 1–4 bytes using a custom variable-length format. Each byte represents a base-253 digit; 0xFE is used as a padding marker.

String Codec

EO strings are encoded by reversing adjacent character pairs and XOR-ing each byte with the complementary value from the other half of the string.

UTF-8 → Windows-1252 Conversion

EO uses Windows-1252 encoding. Characters with no CP1252 mapping are replaced with ?. Try mixing Latin, accented, CJK, emoji, and CP1252-exclusive characters (e.g. € " " ' ').
